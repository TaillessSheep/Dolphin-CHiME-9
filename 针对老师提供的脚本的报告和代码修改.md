## 基线B：Dolphin → Qwen3-ASR

### 流程

`track_00.mp4`（视频+混合音频）→ `Dolphin`（利用人脸唇动提取目标说话人）→ 分离后音频（`s1.mp4`）→ 按 ASD 窗口切分 → `Qwen3-ASR` → 转录 → 评估 → `WER`

### 对老师提供的代码的修改

#### 修改1：`prepare_dolphin_manifest.py` 中的输出文件匹配
- **问题**: 脚本期望 Dolphin 输出 `speaker1_est.wav`，实际输出是 `s1.mp4`。
- **修改**: 在 `run_dolphin_track` 函数中，将 `est = out_dir / "speaker1_est.wav"` 改为 `est = out_dir / "s1.mp4"`。

#### 修改2：添加文件等待逻辑（可能，因为有的时候生成文件了但没被检测到，临时补丁不一定有效）
- **问题**: NFS 存储延迟，文件写入后需要等待才能被检测到。
- **修改**: 添加等待循环，最多等待 15 秒，每秒检查一次文件是否存在。

#### 修改3：增加 `.mp4` 文件的 fallback 查找
- **修改**: 如果 `s1.mp4` 不存在，尝试查找目录下任意 `.mp4` 文件作为替代。

**完整代码示例：**

```python
def run_dolphin_track(
        *,
        dolphin_repo: Path,
        track_video: Path,
        out_dir: Path,
        cuda_device: int,
        detect_every_n: int,
        face_scale: float,
        reuse: bool,
) -> Path:
    """Run Dolphin on a single MCoRec face-crop track and return separated audio."""
    # 使用绝对路径
    out_dir = out_dir.resolve()
    est = out_dir / "s1.mp4"

    if reuse and est.exists() and est.stat().st_size > 0:
        print(f"[reuse] {est}")
        return est

    out_dir.mkdir(parents=True, exist_ok=True)
    inference_py = find_dolphin_inference(dolphin_repo)
    cmd = [
        sys.executable,
        str(inference_py),
        "--input", str(track_video),
        "--output", str(out_dir),
        "--speakers", "1",
        "--detect-every-n", str(detect_every_n),
        "--face-scale", str(face_scale),
        "--cuda-device", str(cuda_device),
    ]
    run_cmd(cmd, cwd=dolphin_repo)

    # 等待文件写入完成（最多等待 15 秒）
    import time
    for _ in range(15):
        if est.exists():
            break
        time.sleep(1)

    if not est.exists():
        # 尝试找其他 mp4 文件
        mp4_files = list(out_dir.glob("*.mp4"))
        if mp4_files:
            est = mp4_files[0]
            print(f"[INFO] Found alternative mp4: {est}")
        else:
            raise RuntimeError(f"Dolphin did not produce s1.mp4 in {out_dir}")

    return est
```

## 当前遇到的问题

### 5.1 ASD 文件格式不兼容（核心问题）
- **问题描述**:
  - CHiME-9 官方提供的 `track_xx_asd.json` 中的数值是负数（如 -2.1, -2.2）。
  - 老师提供的 `common_mcorec.py` 中的 `parse_asd_scores()` 直接返回这些负数值。
  - `active_windows_for_track()` 使用 `score >= threshold(0.5)` 判断，所有负数都 < 0.5。
  - 导致 `active_frames` 为空，回退到使用整个 track（不切分）。
- **受影响的文件**: `common_mcorec.py` 中的 `_score_from_item()` 和 `active_windows_for_track()`。

### 5.2 Dolphin 输出视频的时间戳偏移（推测）
- **问题描述**:
  - Dolphin 输出的 `s1.mp4` 可能重新编码，时间戳重置为 0。
  - 切分时使用的 `rel_start` / `rel_end` 是相对于原始视频的。
  - 导致从 `s1.mp4` 中切出的音频片段时间不对齐，最终 WER 接近 100%。

### 5.3 部分说话人无法检测到人脸
- **问题描述**:
  - 某些说话人（spk_1、spk_5）的视频第一帧检测不到人脸。
  - Dolphin 报错：`ValueError: First frame must detect at least 1 faces, but only found 0`
